1. 목표·타깃·콘텐츠 범위 정하기
할 일
이 페이지로 “누구에게 무엇을 보여주고, 어떤 인상을 주고 싶은지”를 한 문장으로 정리
타깃: 채용 담당자 / 자문·컨설팅 의뢰 / 동료 리서처 / 개인 투자자 등
“반드시 넣을 것” vs “나중에 추가해도 되는 것” 구분
AI 활용
질문 예시
“퀀트 전략·리서치 전문가 개인 웹페이지를 만들려고 해. 타깃은 [OO]. 전문성과 신뢰를 주려면 어떤 섹션을 넣는 게 좋을지 5~7개 추천해줘.”
“퀀트/알고 리서치 포트폴리오 페이지에서 피해야 할 클리셰와, 대신 강조하면 좋은 요소를 알려줘.”
활용법: 답변을 “목표 1문장 + 타깃 + 필수 섹션 리스트”로 정리해 두고, 이후 모든 단계에서 이걸 기준으로 판단
2. 정보 구조(IA) & 와이어프레임
할 일
페이지에 올 섹션 순서 정하기 (예: 소개 → 연구 분야 → 퍼블리케이션/노트 → 방법론/도구 → 연락처)
각 섹션에 “무슨 정보가 들어갈지” 한 줄씩 정의
간단한 와이어프레임(종이/Figma/마크다운으로 상단→하단 블록만)
AI 활용
질문 예시
“[1단계에서 정한 목표·타깃]을 위한 리서치 랜딩 페이지 IA를 만들어줘. 섹션 이름과 각 섹션에 들어갈 내용을 한 줄로만 적어줘.”
“아카데믹/퀀트 리서치 웹페이지에서 ‘연구 분야’와 ‘퍼블리케이션’ 섹션을 어떤 순서로 두는 게 좋을지, UX 관점으로 비교해줘.”
활용법: AI가 제안한 IA를 그대로 쓰기보다, “이건 빼고, 여기에 OO 추가해줘”처럼 수정 지시를 반복해서 본인 버전으로 고정
3. 콘텐츠 초안(텍스트/자료)
할 일
각 섹션별로 실제로 쓸 문장/불릿 초안 작성
논문, 노트, 블로그, 깃허브 등 “넣을 자료 목록”과 링크/썸네일 정리
수치/성과가 있다면 (백테스트 요약, 논문 수, 프로젝트 수 등) 표로 정리
AI 활용
질문 예시
“퀀트 리서치 전문가 소개 문단을 3~4문장으로 써줘. 강조할 키워드: [팩터, 백테스트, 파이썬, …]. 말투는 겸손하지만 전문적으로.”
“이 목록을 웹페이지 ‘연구 분야’ 섹션용 불릿 포인트로 바꿔줘: [본인이 적은 리스트].”
활용법: 생성된 문장을 그대로 쓰지 말고, “숫자/구체 사례만 내 걸로 바꾸고, 톤만 유지해줘”라고 하면 전문성 유지에 도움
4. 디자인 방향 & 스타일 가이드
할 일
“참고할 사이트 2~3개” 정하기 (해외 퀀트 펀드, 대학 리서치 랩, 알고 트레이딩 블로그 등)
색상·폰트·여백·카드/섹션 스타일을 짧게 정리 (현재 링크바이오 화면설계서처럼 “키워드 + 1줄 설명”만 있어도 됨)
AI 활용
질문 예시
“퀀트/핀테크 리서치 웹페이지에서 전문성과 신뢰를 주는 색상 조합 2가지와, 피해야 할 조합을 알려줘. hex 코드 포함.”
“데이터/리서치 중심 웹페이지에서 가독성이 좋은 폰트 조합 (헤딩 + 본문) 추천해줘. Google Fonts 기준.”
“[참고 URL 또는 스크린샷 설명] 같은 분위기의 웹페이지를 만들려고 해. CSS/테일윈드로 그 무드를 내는 키워드 5개만 알려줘.”
활용법: 답변을 “색상 팔레트 / 폰트 2종 / 여백·radius 규칙” 정도로 문서에 적어 두고, 다음 단계에서 “이 가이드에 맞춰 구현해줘”라고 할 때 사용
5. 기술 스택 & 페이지 구현
할 일
리서치 페이지를 “같은 도메인 하위 경로”로 둘지(예: jaekim.com/research), 별도 서비스로 둘지 결정
선택한 스택(예: 지금처럼 Next.js + React)으로 페이지 라우트 추가, 2~4단계에서 정한 IA·콘텐츠·스타일 반영
AI 활용
질문 예시
“Next.js 앱 라우터에서 /research 페이지를 추가하고, [화면설계서 또는 IA] 이 구조로 컴포넌트를 나눠줘. 파일 구조만 제안해줘.”
“이 섹션은 [마크다운/JSON] 콘텐츠를 그대로 넣을 거야. React 컴포넌트로 렌더링하는 코드만 작성해줘.”
“[4단계 스타일 가이드]에 맞춰서 이 블록의 테일윈드 클래스를 전문적이고 차분한 톤으로 바꿔줘.”
활용법: 한 번에 전체 페이지를 부탁하기보다, “헤더 → 소개 → 연구 분야 → 푸터”처럼 섹션 단위로 구현 요청하고, 각 요청 시 IA 1줄 + 스타일 키워드를 같이 주면 일관성 있게 나옴
6. 검수·배포·유지보수
할 일
데스크톱/모바일에서 한 번씩 훑어보기, 링크·오타 점검
SEO: 타이틀, 설명, OG 이미지 한 번 설정
배포 후 “리서치 링크” URL을 실제 리서치 페이지 주소로 변경 (지금 profile-data.ts의 research URL 수정)
AI 활용
질문 예시
“이 HTML/React 마크업에서 접근성 문제 될 만한 부분만 짧게 짚어줘.”
“퀀트 리서치 개인 페이지용 메타 타이틀과 description 2안 써줘. 60자, 160자 제한.”
활용법: “전문성·신뢰”를 다시 한 번 체크하고 싶다면, “이 페이지를 채용 담당자/동료 리서처가 10초만 볼 때 어떤 인상일지 요약해줘”라고 하면 피드백용 요약을 얻을 수 있음
한 줄 요약
순서	할 일	AI 활용 포인트
1	목표·타깃·콘텐츠 범위	“퀀트 리서치 전문가 페이지에 넣을 섹션 추천” + 피할 클리셰
2	IA & 와이어프레임	IA 초안 생성 → “빼고/추가해줘”로 수정
3	콘텐츠 초안	소개문·불릿 초안 작성, 톤/숫자만 본인 것으로 수정
4	디자인·스타일	색상/폰트/분위기 키워드 추천 → 가이드 문서화
5	구현	섹션 단위로 구현 요청 + IA·스타일 문맥 함께 전달
6	검수·배포	접근성·메타·10초 인상 요약으로 마무리
지금 프로젝트는 Next.js 기반 링크바이오가 있으니, 리서치 페이지는 같은 저장소에 /research(또는 /[slug]/research) 같은 라우트를 추가해 두고, profile-data.ts의 research URL만 그 경로로 바꾸는 방식이 관리하기 가장 쉽습니다.
원하면 2단계(IA)나 5단계(파일 구조/코드 초안)를 지금 문서 구조에 맞춰 더 구체적으로 풀어줄 수 있어요.
