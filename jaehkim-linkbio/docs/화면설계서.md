## 0. 문서 메타

- 문서명: Link-in-Bio 공개 페이지 화면설계서 v1
- 대상: 디자이너/프론트엔드/AI 코드 생성 도구
- 페이지 수: 1 (사용자 공개 프로필 페이지)
- URL 정책: `/{slug}` (아이디 슬러그)
  - 예: `/jaekim`

---

## 1. 서비스 목적 & 톤

### 목적

- 사용자가 하나의 페이지에서 자신의 외부 채널 링크를 공유
- 문의(리드) 수집을 이메일 전송 방식으로 처리

### 디자인 컨셉 키워드

- 세련됨 / 현대적 / 단순함 / “명품(럭셔리) 스타일” / 트렌디

### 스타일 가이드(결정안)

- **베이스 컬러**: 거의 블랙(차콜) + 소프트 화이트
- **포인트 컬러**: 메탈릭 실버 느낌의 중립 톤(회백) + 아주 미세한 골드/샴페인 하이라이트(과하지 않게)
- **타이포**: 산세리프(매우 미니멀, 자간 살짝 여유)
- **형태**: 둥근 모서리(large radius), 넓은 여백, 얇은 라인/보더, 은은한 그림자(낮은 opacity)
- **질감**: 단색 배경 + 아주 약한 그라데이션/노이즈(고급스러운 무드)

> 핵심: “꾸밈은 최소, 재질감은 고급” (명품 UI의 느낌)

## 2. IA(정보구조)

### 2.1 공개 페이지(`/{slug}`)

1. 헤더 영역
2. 프로필 영역(닉네임)
3. 링크 버튼 리스트(4개)
4. 문의하기 버튼
5. 하단 정책/안내 문구
6. 다크/라이트 모드 토글(우상단 또는 하단)

---

## 3. 화면 구성(레이아웃)

### 3.1 공통 레이아웃 규칙

- 페이지 전체는 **중앙 정렬 컨테이너**(카드형 레이아웃)
- 최대 폭: 데스크탑에서 너무 넓지 않게(가독성 유지), 모바일은 풀폭
- 위아래 여백 넉넉히(럭셔리 무드)

### 3.2 데스크탑(>=1024px) 레이아웃

- 배경: 전체 화면 단색(다크면 거의 블랙, 라이트면 소프트 화이트)
- 중앙 카드(컨테이너):
  - 폭: 420~520px 권장
  - 카드 내부 상단에 프로필, 그 아래 링크 버튼

- 우상단(뷰포트 기준) 또는 카드 우상단에 **모드 토글** 배치

### 3.3 모바일(<1024px) 레이아웃

- 컨테이너 폭: 100% - 좌우 패딩
- 버튼은 풀폭
- 토글은 상단 우측 고정 또는 카드 상단 우측

---

## 4. 컴포넌트 상세 명세

### 4.1 헤더/프로필 영역

- 요소:
  - **닉네임 텍스트(필수)**: slug 사용자 닉네임
  - (선택) 아주 짧은 서브텍스트(없어도 됨): “Links” 같은 라벨을 미세하게

- 타이포:
  - 닉네임: Bold 또는 SemiBold, 큰 사이즈

- 정렬:
  - 중앙 정렬

**데이터 필드**

- `displayName` (string, required)

---

### 4.2 링크 버튼 리스트(4개)

링크 종류:

1. 오픈카톡
2. 유튜브(채널 링크)
3. 쓰레드
4. 링크드인

**버튼 공통 규격**

- 형태: 라운드(큰 radius), 풀폭
- 구성: 좌측 아이콘 + 텍스트 라벨 + 우측 “↗”(새 탭) 미니 아이콘(선택)
- 인터랙션:
  - Hover: 밝기/보더 변화 + 살짝 상승(1~2px)
  - Active: 눌림(translateY)
  - Focus: 키보드 포커스 링(접근성)

- 클릭:
  - `target="_blank"` + `rel="noopener noreferrer"`

**라벨(권장)**

- “Open KakaoTalk”
- “YouTube”
- “Threads”
- “LinkedIn”

**데이터 필드**

- `links.openKakao` (url)
- `links.youtubeChannel` (url)
- `links.threads` (url)
- `links.linkedin` (url)

**에러/비활성 규칙**

- 해당 링크가 없으면:
  - (옵션 A) 버튼 숨김
  - (옵션 B) 비활성(disabled) + “준비중” 표시

- v1 권장: **숨김**(페이지를 더 깔끔하게)

---

### 4.3 다크/라이트 모드 토글

- 정책: **시스템 설정 기본 + 토글로 오버라이드**
- 저장: 로컬스토리지(브라우저) 권장
- UI 형태:
  - 아주 미니멀한 토글(아이콘: 달/해)
  - 배치: 카드 우상단 또는 뷰포트 우상단

**상태**

- `themeMode`: `"system" | "light" | "dark"`
- 토글 동작:
  - 기본은 system
  - 사용자가 토글을 누르면 light<->dark로 고정
  - (선택) 길게 누르거나 메뉴로 system 복귀 제공
  - v1 단순화: 토글 누르면 “system 기준 현재값”에서 반대 모드로 고정, 설정에서 system 복귀는 추후

---

### 4.4 문의하기 버튼 + 모달(핵심)

#### 문의하기 버튼

- 라벨: “Contact”
- 스타일: 링크 버튼보다 “더 중요한 CTA” 느낌
  - 보더/배경 대비를 더 주되, 과한 색은 피하기(샴페인/실버 톤으로 강조)

#### 모달 UI

- 트리거: 문의하기 클릭
- 모달 구성:
  - 타이틀: “Contact”
  - 설명 문구(짧게): “Leave your message and I’ll get back to you.”
  - 입력 필드 3개:
    1. Name (필수)
    2. Email (필수)
    3. Message (필수, textarea)

  - 버튼 2개:
    - Cancel(좌) / Send(우, primary)

- 닫기:
  - X 버튼(우상단)
  - 배경 클릭 닫기(가능)
  - ESC 닫기(키보드)

#### 유효성(Validation)

- Name: 2~30자(공백만 금지)
- Email: 이메일 형식 검사
- Message: 5~1000자
- 에러 표기:
  - 필드 하단에 한 줄 텍스트(작고 깔끔)
  - 빨간색 대신 “톤 다운된 경고 컬러”(명품 톤 유지)

#### 전송 동작(이메일 전송)

- Send 클릭 시 상태:
  1. Loading: 버튼 로딩 스피너, 입력 비활성
  2. Success: **토스트** “Sent successfully” + 모달 자동 닫기
  3. Failure: 모달 하단에 에러 메시지(재시도 가능)

#### 스팸 방지(권장 최소)

- “허니팟(hidden field)” 1개 + 간단한 레이트리밋(서버)
- 캡차는 v1에서 생략 가능(UX 저하)

**API 요구사항(프론트 기준)**

- `POST /api/contact`
  - body: `{ name, email, message, slug, pageUrl }`
  - response:
    - 200: success
    - 400: validation error
    - 429: rate limit
    - 500: server error

- 이메일 전송:
  - 수신자: 서비스 운영자 이메일(고정)
  - 제목 예: `[LinkBio Contact] {slug} - {name}`
  - 본문에 name/email/message/pageUrl 포함

---

### 4.5 하단 정책/안내 문구

- 위치: 페이지 최하단, 작은 글씨
- 문구 예시:
  - “By submitting, you agree your email may be used to respond to your inquiry.”

- (선택) “Privacy” 링크는 추후 확장 가능

## 5. 상태 정의(프론트 상태 머신 요약)

1. 페이지 로드

- 상태: `loading -> ready`
- 실패 시: `error` (간단 메시지 + 새로고침 안내)

2. 테마

- `system/light/dark`
- 초기값: `system`
- 사용자 오버라이드 시 `localStorage` 저장

3. 모달

- `closed | open`
- 폼 상태:
  - `idle | validating | submitting | success | error`

## 6. 접근성(A11y) 체크리스트(필수)

- 모든 버튼/토글/입력은 키보드 탭 이동 가능
- 포커스 링 표시(테마에 따라 대비 유지)
- 모달 열릴 때 포커스는 타이틀/첫 입력으로 이동
- 모달 열림 동안 배경 스크롤 잠금
- 모달 닫힐 때 포커스는 “문의하기 버튼”으로 복귀

## 9. 구현 우선순위(MVP → V1+)

### MVP(이번 화면설계서 기준)

- 1페이지 공개 프로필
- 4개 링크 버튼(없으면 숨김)
- 다크/라이트 토글(시스템+오버라이드)
- 문의하기 모달 + 이메일 전송
- 하단 안내 문구

### V1+ (다음 버전)

- 편집 페이지(로그인/관리)
- 링크 순서 드래그 앤 드롭
- 커스텀 라벨/아이콘 선택
- analytics(클릭 수, 문의 수)
